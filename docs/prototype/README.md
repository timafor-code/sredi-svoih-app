# Prototype files

Здесь лежат HTML/JavaScript-прототипы приложения «Среди Своих».

## sredi-svoih.html

Главный HTML/React-прототип приложения.

Используется как основной источник правды для:
- структуры экранов;
- нижней навигации;
- главной страницы;
- страницы молитв и зманим;
- страницы событий;
- страницы контактов;
- страницы профиля;
- вложенных экранов профиля;
- карточки контакта;
- модального окна Омера;
- визуального dark glass стиля.

При переносе в React Native запрещено использовать WebView или HTML.

Нужно реализовать нативные React Native компоненты.

## ios-frame.jsx

Дополнительный визуальный reference.

Не переносить fake iPhone frame, Dynamic Island и fake status bar в production-приложение.


# Prototypes

## admin-events-center.html

HTML/JavaScript-прототип web-админки Admin Events Center для проекта «Среди Своих».

Используется как source of truth для будущей production web-админки:

- структура разделов админки;
- sidebar/topbar navigation;
- dashboard;
- управление событиями;
- import review flow;
- ручное создание и редактирование событий;
- конструктор вариантов участия и оплаты;
- mock-сценарии внутренней платной регистрации;
- выбор количества вариантов участия;
- расчёт итоговой суммы и мест;
- управление регистрациями;
- участники;
- приглашения;
- настройки;
- future-разделы: контакты общины, уведомления, медиа, расписание молитв, отчёты, журнал действий;
- визуальный dark glassmorphism style, согласованный с текущим мобильным приложением.

Прототип не является production-кодом.

Ограничения:

- не подключается к Supabase;
- не использует реальные backend-запросы;
- не использует серверные ключи повышенных прав;
- не использует серверные административные методы Supabase в браузере;
- использует mock data внутри HTML/JS;
- открывается локально в браузере как обычный HTML-файл.

Production-реализация должна использовать Supabase Auth, RLS и защищённые RPC.
